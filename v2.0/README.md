# Vexilon – Local AI Chrome Assistant (v2.0)

Vexilon v2.0 is a locally powered AI browser assistant built using Ollama, Flask, and a Chrome extension interface.

This version focuses on running conversational AI directly on local hardware using Ollama-hosted language models such as `llama3`, while integrating AI interaction directly into the browser through a floating assistant UI.

Unlike the earlier cloud-based version, v2.0 moves toward local inference workflows and self-hosted AI experimentation.

---

# Core Features

## Local AI Execution
- Runs entirely on local hardware
- Powered by Ollama
- No external AI API dependency required

## Multi-Model Support
Supports Ollama-compatible models including:
- llama3
- mistral
- gemma

## Browser-Integrated Assistant
- Floating draggable AI chat interface
- Injected directly into webpages
- Lightweight Chrome extension workflow

## Runtime Conversational Context
- Maintains active conversation history during runtime
- Context-aware responses within active sessions

## Backend Token Stream Processing
- Uses streamed token generation from Ollama
- Aggregates generated tokens before returning final response

## OpenAI-Compatible Backend Structure
- Backend implemented using OpenAI Python SDK compatibility
- Easily extendable for future model integrations

# Verified Implementation Details

## Floating Browser Assistant
The extension injects a draggable floating chat interface directly into webpages using `content.js`.

## Manifest V3 Configuration
The extension uses:
- Manifest V3
- content scripts
- localhost backend permissions
- web accessible resources

## Ollama Integration
The backend connects to Ollama through:

```python
base_url="http://localhost:11434/v1"
```

using the OpenAI SDK compatibility layer.

## Runtime Chat Context
Conversation history is temporarily maintained in memory during active backend runtime.

## Local Response Workflow
User prompts are:
1. Sent from the extension
2. Processed by Flask
3. Forwarded to Ollama
4. Generated by the local model
5. Returned to the browser assistant

# Setup Instructions

## 1. Install Ollama

Download Ollama from:

https://ollama.com

---

## 2. Run a Local Model

Example:

```bash
ollama run llama3
```

You can also use:
- mistral
- gemma
- codellama

---

## 3. Install Backend Dependencies

Navigate to the backend folder:

```bash
cd PY
```

Install required packages:

```bash
pip install flask flask-cors openai
```

---

## 4. Start Flask Backend

```bash
python app.py
```

Default backend server:

```text
http://localhost:5000
```

---

## 5. Load Chrome Extension

1. Open:

```text
chrome://extensions/
```

2. Enable **Developer Mode**

3. Click:

```text
Load unpacked
```

4. Select the `v2.0/` folder

# Current Limitations

The current implementation is an early-stage prototype and has several limitations:

- No persistent database
- No authentication system
- Runtime-only memory
- No WebSocket implementation
- No Docker support
- No CI/CD pipeline
- No dedicated frontend framework
- No session management
- Single-user workflow
- No production deployment infrastructure
